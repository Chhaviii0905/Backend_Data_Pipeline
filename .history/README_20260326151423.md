Backend Data Pipeline

This project is a containerized backend data pipeline built to demonstrate how data moves between services and gets stored in a database using an upsert strategy.

It simulates a real-world scenario where data is fetched from an external API, processed, and loaded into PostgreSQL.

The entire system runs using Docker and docker-compose.

Architecture Overview

The system consists of three services:

Mock API (Flask)
Serves customer data from a local JSON file with pagination support.
Pipeline Service (FastAPI + DLT)
Fetches data from the mock API and loads it into PostgreSQL using merge (upsert) logic.
PostgreSQL
Stores the ingested customer data.

All services are containerized and orchestrated using docker-compose.

Running the Project
Prerequisites
Docker Desktop (running)
docker-compose
Start All Services

docker-compose up --build -d

This starts:

PostgreSQL (port 5432)
Flask mock server (port 5000)
FastAPI pipeline service (port 8000)

To verify containers are running:

docker-compose ps

API Endpoints
Mock Server (Flask – Port 5000)

Health check:
GET http://localhost:5000/api/health

Get paginated customers:
GET http://localhost:5000/api/customers?page=1&limit=5

Pipeline Service (FastAPI – Port 8000)

Swagger UI:
http://localhost:8000/docs

Trigger ingestion:
POST http://localhost:8000/api/ingest

Fetch customers from database:
GET http://localhost:8000/api/customers?page=1&limit=5

Get single customer:
GET http://localhost:8000/api/customers/{customer_id}

Data Loading Logic

The pipeline uses dlt with:

write_disposition = "merge"
primary_key = "customer_id"

This ensures:

Existing records are updated
New records are inserted
Duplicate records are avoided
Stopping Services

docker-compose down
